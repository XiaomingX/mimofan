# 模式与审批

mimofan 有两个相关概念：
- **TUI 模式**：交互方式（Plan / Agent / YOLO）
- **审批模式**：工具执行前的确认策略

## TUI 模式

按 `Tab` 切换：**Plan → Agent → YOLO → Plan**

| 模式 | 说明 |
|------|------|
| **Plan** | 设计优先。只读工具可用，禁止 shell 和文件写入 |
| **Agent** | 多步工具使用。shell 需配置 `allow_shell = true`，每次调用需审批 |
| **YOLO** | 全自动。启用 shell + 信任模式，所有工具自动通过。仅限可信仓库 |

### 工具可用性

| 工具类型 | Plan | Agent | YOLO |
|---------|:----:|:-----:|:----:|
| 只读文件/搜索/诊断 | ✅ | ✅ | ✅ |
| 文件写入/补丁 | ❌ | ✅ | ✅ |
| Shell 命令 | ❌ | 需审批 | ✅ |
| 付费/外部服务 | 需审批 | 需审批 | 自动通过 |

## 审批模式

通过 `/config` 修改 `approval_mode`：

| 模式 | 说明 |
|------|------|
| `suggest`（默认） | 按 TUI 模式规则决定 |
| `auto` | 自动审批所有工具 |
| `never` | 阻止所有非只读工具 |

## Esc 键行为

`Esc` 是取消栈，不是模式切换：
1. 先关闭弹出菜单
2. 取消正在运行的请求
3. 丢弃排队的草稿
4. 清空当前输入

## 信任模式

默认文件工具限制在工作区目录内。启用信任模式允许访问外部文件：

```
/trust
```

YOLO 模式自动启用信任。

## CLI 常用参数

| 参数 | 说明 |
|------|------|
| `-p, --prompt <TEXT>` | 单次提示模式 |
| `--model <MODEL>` | 指定模型 |
| `--workspace <DIR>` | 工作区目录 |
| `--yolo` | YOLO 模式启动 |
| `-r, --resume <ID>` | 恢复会话 |
| `-c, --continue` | 继续最近会话 |
| `--profile <NAME>` | 选择配置 profile |
